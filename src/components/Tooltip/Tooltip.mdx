# Tooltip

A small floating label that explains a control on hover or focus. Built on Radix (hover +
focus triggers, keyboard, ARIA, collision handling) and styled to the Swiss Army Figma вЂ” a
slate surface with `body/sm` text.

## When to use

- **Clarify an unlabelled control** вЂ” icon-only buttons, truncated text, a terse field.
- **Short, supplementary hints** вЂ” not essential information (tooltips are hidden until hover
  and unavailable on touch). For required info, use visible text.
- Keep it brief: the surface caps at **256px** wide and wraps.

## Position

`side` (Figma `positioned`: top / right / bottom / left) and `align` (Figma `placement`:
start / center / end) set where the tooltip appears. Radix flips `side` automatically in RTL
and on collision, and keeps an **8px gap from the viewport edge** (`collisionPadding`).

## Motion & timing

Opens with a **GROW** transition (scale + fade from the side facing the trigger) over
`--duration-fast`. Show delay is **100ms** (`delayDuration`).

## Accessibility

- **Triggers** вЂ” opens on hover *and* keyboard focus; `Esc` dismisses. Radix wires
  `aria-describedby` from the trigger to the content.
- **Not for essential info** вЂ” tooltips don't appear on touch and vanish on blur; never put
  required content (or interactive elements) inside one.
- **Icon-only triggers** still need their own `aria-label` вЂ” the tooltip describes, it doesn't
  name the control.

## Token-driven styling

Surface is `background/accent/gray/bolder`, text `text/oncolor/default`, padding `--space-8`,
radius `--radius-2`, type `body/sm` вЂ” all semantic tokens. The only raw value is the 256px max
width, which has no design token.
